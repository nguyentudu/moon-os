# Boundary Problem Hypotheses v1

## Purpose

Separate evidence-supported hypotheses from unsupported certainty.

## Hypothesis Ranking

### 1. UI / Instruction Bias

Likelihood: high

Evidence:

- UI centers Decision language.
- UI does not define Not Decision inline.
- UI does not provide Not Decision examples.
- Decision is the first button.

### 2. Weak or Ambiguous Not Decision Excerpts

Likelihood: high

Evidence:

- Several expected Not Decision excerpts are decision-adjacent.
- Reviewer comments cluster around missing rationale, unclear intent, and why questions.

### 3. Overly Broad Decision Definition

Likelihood: moderate

Evidence:

- Reviewers treated many Moon-related statements as decisions.
- However, the formal definition already contains limiting criteria.

### 4. Reviewer Misunderstanding

Likelihood: moderate

Evidence:

- Reviewers used Unclear when confused, but never used Not Decision.
- This may indicate they did not understand when Not Decision applies.

## Most Likely Combined Explanation

The most likely cause is not a single issue.

The strongest hypothesis is a combination of UI/instruction bias plus ambiguous Not Decision excerpts.

The definition may need clearer operational examples later, but should not be rewritten before Round 2 evidence.
