# Benchmark Limitations v1

## Purpose

Document limitations for the Time To Context benchmark and prevent overclaiming.

## Current Evidence

Raw TTC logs were imported from chat.

Source:

chat-provided TTC logs

## Participant Identity

unknown

## No Independent Reviewer Proof

Reviewer independence is unknown.

Therefore:

- no market proof exists
- no independent reviewer proof exists
- no broad usability proof exists
- no validated economic proof exists

## Same Participant Rule

If these runs came from the same participant, results must be treated as:

single-participant evidence

Current status:

same participant unknown

## Sample Size Rule

The imported logs include 3 Memory Mode runs and 3 Manual Mode runs.

Because participant identity is unknown, this should not be described as 3 independent participants.

## Mode Contamination Risk

The ordering appears to show Memory Mode runs before Manual Mode runs.

If the same person completed both modes, Manual Mode may be contaminated by prior exposure to Decision Memory answers.

## Duplicate / Malformed Entry Risk

Memory Mode Q3 contains duplicate malformed entries in all three Memory runs.

Primary timing analysis excludes these duplicate malformed entries.

## Scope Limitation

The benchmark uses Moon's own repository as the dataset.

It does not prove performance in external domains.

## Claim Boundary

Allowed claim:

early TTC signal

Disallowed claims:

- market proof
- independent reviewer proof
- validated ROI
- universal productivity improvement
- proof that Moon works for all domains
