# Participant Instructions v1

## Purpose

These instructions are for participants in the Time To Context benchmark.

## What You Are Testing

You are testing how quickly and confidently someone can answer why Moon OS exists the way it does.

## Conditions

You may be assigned to one of two conditions:

Group A:

Use the repository manually.

Group B:

Use Decision Memory materials.

## Rules

- Answer honestly.
- Do not guess if evidence is missing.
- Record time spent per question.
- Cite files or decision records used.
- Mark uncertainty clearly.
- Do not use external sources.

## Questions

Answer the 10 benchmark questions provided in your task file.

## Ratings

For each answer, rate:

- answer completeness
- confidence
- perceived difficulty

Use the scoring rubric.
