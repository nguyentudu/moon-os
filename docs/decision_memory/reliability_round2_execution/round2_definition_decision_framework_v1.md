# Round 2 Definition Decision Framework v1

## Purpose

Document future decision paths for the Decision Definition after Round 2 evidence exists.

This file does not change the Decision Definition.

## Path A: Keep Definition

Choose this path only if Round 2 evidence shows:

- reviewers use Decision, Not Decision, and Unclear in expected ways
- Not Decision appears naturally on clear non-decision excerpts
- boundary cases decrease or become explainable
- reviewer comments no longer show recurring confusion about metrics/status/taxonomy

Allowed conclusion:

The existing definition may be operational enough with improved instructions and examples.

Not allowed:

Claiming the definition is validated for all future extraction.

## Path B: Revise Definition

Choose this path only if Round 2 evidence shows:

- Not Decision remains absent or rare
- reviewers continue to classify metrics/status/taxonomy as Decision
- comments show confusion about what counts as a choice
- Unclear remains overused as the only non-Decision escape hatch

Allowed conclusion:

The definition may need a narrower operational boundary.

Not allowed:

Rewriting the definition without a documented revision proposal and evidence review.

## Evidence Required Before Any Definition Decision

- imported raw reviewer submissions
- pairwise agreement analysis
- Not Decision usage analysis
- disagreement excerpt list
- boundary cluster analysis
- reviewer comment analysis
