# Time To Context Test Packet v1

## Purpose

Prepare a future human test comparing manual repository reading against Decision Memory answers.

This packet does not record results.

## Test Inputs

Manual condition:

- Give reviewer the Moon OS repository.
- Ask the 10 benchmark questions.
- Allow normal manual reading and search.

Decision Memory condition:

- Give reviewer the 10 benchmark questions.
- Give reviewer:
  - ../decision_memory_examples_v1.md
  - decision_query_answer_set_v1.md
  - decision_query_trace_matrix_v1.md

## Benchmark Questions

1. Why did Moon build Constitutional Core before UI?
2. Why did Moon delay data ingestion?
3. Why did Moon create Trust QA?
4. Why is Today not a dashboard?
5. Why is Runtime Demo synthetic?
6. Why is Time To Context the next KPI?
7. Why not start with supply chain?
8. Why is Moon not just a GPT wrapper?
9. Why does Moon need Decision Memory?
10. Why is human evidence required?

## Data To Collect In Future Test

For each question:

- time to first credible answer
- files opened
- answer confidence
- evidence links cited
- missing context noted
- reviewer comments

## Scoring Guidance

Do not score by whether the answer is flattering to Moon.

Score by:

- source support
- clarity
- completeness
- uncertainty visibility
- speed

## No Fake Result Rule

This packet contains no measured outcomes.

A future test must collect real timing and reviewer evidence before claiming Time To Context reduction.
