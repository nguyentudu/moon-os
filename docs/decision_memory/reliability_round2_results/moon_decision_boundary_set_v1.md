# Moon Decision Boundary Set v1

## Purpose

Capture the operational boundary evidence from Round 1 and Round 2.

This is not a Decision Definition rewrite.

## Current Boundary Set

### Decision Signals

Classify as Decision when an excerpt itself contains or clearly implies:

- a selected path
- a constraint on Moon behavior
- a claim boundary
- a governance rule
- rationale or consequence

Examples from existing excerpts:

- EV-006: Metrics inform. Metrics do not decide.
- EV-009: Trust QA may block surfacing even when awareness relevance is high.
- EV-013: No real data is used yet because the demo avoids ingestion risk.
- EV-021: Claim only early TTC signal.

### Not Decision Signals

Classify as Not Decision when an excerpt is only:

- a metric value
- a count
- a status check
- a data flag
- a UI question
- a taxonomy item
- an observation

Examples with Round 2 evidence:

- EV-024: 5 demo cases visible: yes.
- EV-026: Memory average time: 99.4 seconds.
- EV-027: Memory average confidence: 2.9.

### Unclear Signals

Classify as Unclear when the excerpt is decision-shaped but missing local rationale or consequence.

Examples:

- EV-029: C-030 Deploy TTC Lab publicly.
- EV-031: response received: no.
- EV-034: Why did Moon delay data ingestion?

## Evidence Boundary

This boundary set is provisional.

It summarizes observed evidence and should guide Round 3 or a definition review. It does not authorize large-scale Decision Memory expansion.
