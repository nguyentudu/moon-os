# Reasoning Case Set v1

## Case 1 — Decision Expansion Gate

### Context

Boundary Validation completed.

Definition survived testing.

Decision Expansion remains blocked.

### Question

Should Decision Expansion be opened now?

Why?

### Required Answer Elements

- rationale
- evidence
- risks
- recommendation
- uncertainty

### Why This Is Novel

The participant must integrate boundary validation, reliability evidence, and expansion risk. No single repository document directly answers this future gate question.

## Case 2 — Round 3 Design

### Question

If you were Moon Architect, how would you design Reliability Round 3?

### Required Answer Elements

- purpose
- evidence requirements

- success criteria
- failure criteria

### Why This Is Novel

The participant must synthesize from Round 1, Round 2, boundary audit, and definition validation artifacts to design a future study.

## Case 4 — Boundary Interpretation

### Scenario

A new reliability round shows reviewers agree on clear Decision examples, but still disagree on cases where an excerpt includes intention without consequence.

### Questions

What does this evidence imply?

What should Moon do next?

### Required Answer Elements

- interpretation
- likely boundary issue
- risk if ignored
- next action
- uncertainty

### Why This Is Novel

The participant must interpret a new evidence pattern and recommend a next validation move.
