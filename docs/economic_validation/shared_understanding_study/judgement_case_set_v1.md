# Judgement Case Set v1

## Purpose

Judgement cases evaluate whether participants can reach defensible recommendations from incomplete but structured evidence.

## Judgement Case A — Open or Hold Decision Expansion

Prompt:

Moon has better boundary evidence than before, but Not Decision and Unclear cases still create disagreement.

Should Moon open Decision Expansion, hold it, or run another validation step?

Required:

- recommendation
- evidence used
- tradeoffs
- risk of moving too early
- risk of waiting too long

## Judgement Case B — Prioritize Shared Understanding or Time To Contribution

Prompt:

TTC evidence shows participants answer faster with Decision Memory. Blinded quality review shows an early quality signal. Shared understanding is not confirmed.

Should Moon run a Shared Understanding Study first, or move directly to Time To Contribution?

Required:

- recommendation
- rationale
- validation risk
- evidence gap
- next study design note

## Judgement Case C — Interpret Shared Retrieval

Prompt:

Two participants produce nearly identical answers with correct source paths after using Decision Memory.

Does this prove shared understanding?

What should Moon conclude?

Required:

- distinction between retrieval and understanding
- what evidence is missing
- what result would strengthen the claim
- recommendation

## Judgement Case D — Handling a Strong but Small TTC Signal

Prompt:

Decision Memory participants are much faster than manual participants, but sample size is small and all participants used Vietnamese.

What claim is allowed?

What claim is not allowed?

Required:

- allowed claim
- forbidden claims
- reasoning
- next evidence requirement
