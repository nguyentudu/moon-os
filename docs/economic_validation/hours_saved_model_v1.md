# Hours Saved Model v1

## Purpose

Define a simple hours-saved model for future economic validation.

## Formula

hours saved = baseline time - Moon-assisted time

## Required Inputs

- baseline task time
- Moon-assisted task time
- participant count
- task count
- answer quality score
- task comparability notes
- participant role or cost assumption, if making business-case claims

## Quality Adjustment

Hours saved should not count when:

- answer quality is Poor
- evidence is missing
- participant repeats a rejected decision
- participant misunderstands current validity

## Example Use

If a contributor spends 90 minutes manually finding decision context and 20 minutes with Decision Memory, raw time saved is 70 minutes.

This is not economic value until quality and cost assumptions are documented.

## Claim Boundary

Do not convert hours saved into money without:

- participant role
- credible hourly cost or opportunity cost
- sample size
- repeatability
- quality threshold
