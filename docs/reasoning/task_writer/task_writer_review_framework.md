# Task Writer Review Framework v1

## Purpose

The Task Writer Review Framework defines how approved awareness is translated into an awareness artifact.

This is a review structure only. It is not UI, notification, reminder, or action logic.

## Review Questions

### What is being communicated?

Identify the approved awareness item and artifact type.

### Why does it matter?

Explain the continuity significance.

### Why now?

Explain the timing relevance.

### What evidence exists?

List the source-linked evidence that supports the artifact.

### What confidence exists?

State confidence or confidence caveats.

### What uncertainty exists?

Expose missing context, ambiguity, or limits.

### What trust constraints apply?

Include Trust QA outcome and required trust notes.

### Can the user understand it?

Confirm that the artifact is clear, concise, non-coercive, and explainable.

## Review Output

A Task Writer review should include:

- artifact type
- title
- summary
- why it matters
- why now
- evidence
- confidence
- suggested reflection
- optional actions
- trust notes
- uncertainty notes

## Non-Implementation Boundary

This framework does not define UI, rendering, notifications, reminders, storage, ranking, or task execution.
