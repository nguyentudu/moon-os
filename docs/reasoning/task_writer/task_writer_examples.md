# Task Writer Examples v1

## Purpose

These examples illustrate how Moon Task Writer may translate approved awareness into artifacts.

Examples only. No implementation is defined.

## Case 1: Project Alpha

### Input

- Continuity Director: HIGH_CONCERN
- Awareness QA: PRIORITIZE
- Trust QA: ALLOW
- Subject: Project Alpha

### Output

Artifact type: Project Momentum Alert

Title:

Project Alpha may be losing momentum

Summary:

Project Alpha appears connected to an active quarterly goal, but recent activity is absent.

Why It Matters:

Linked to active quarterly goal.

Why Now:

No meaningful activity for 18 days.

Evidence:

Project records and activity history.

Confidence:

Moderate.

Suggested Reflection:

Is Project Alpha still active, paused, or ready for closure?

Optional Actions:

- Review project status.
- Confirm current owner.
- Capture next step or closure note.

Trust Notes:

No additional trust caveat.

Uncertainty Notes:

Absence of activity may reflect an intentional pause.

## Case 2: Relationship Context

### Input

- Continuity Director: CONCERN
- Awareness QA: SURFACE
- Trust QA: ALLOW_WITH_CAUTION
- Subject: relationship context

### Output

Artifact type: Relationship Watch

Title:

A relationship thread may need gentle review

Summary:

There is a permitted relationship continuity concern, but it should be presented with care.

Why It Matters:

Relationship context may affect commitments, collaboration, or personal continuity.

Why Now:

Awareness QA found current relevance.

Evidence:

Approved relationship context and source-linked continuity evidence.

Confidence:

Moderate.

Suggested Reflection:

Is this relationship context something you want Moon to keep lightly in view?

Optional Actions:

- Do nothing.
- Reconfirm the boundary for this relationship context.
- Add a note if the context has changed.

Trust Notes:

Trust QA allowed surfacing with caution. Avoid sensitive details unless the user invites them.

Uncertainty Notes:

Relationship context can be incomplete; treat this as awareness, not a conclusion.

## Example Governance

These examples do not create tasks, reminders, notifications, actions, UI, or decisions on behalf of the user.
