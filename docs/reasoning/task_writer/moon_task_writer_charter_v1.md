# Moon Task Writer Charter v1

## Purpose

Moon Task Writer is the final authority before information reaches Moon Today.

It does not determine continuity significance. That belongs to the Continuity Director.

It does not determine attention priority. That belongs to Awareness / Relevance QA.

It does not determine consent. That belongs to Trust & Consent QA.

It exists to answer:

How should this be presented to the user?

## Core Principle

The Task Writer does not decide what matters.

The Task Writer does not decide what matters now.

The Task Writer does not decide what is permitted.

The Task Writer translates approved awareness into understandable awareness.

## Authority

Task Writer may:

- translate approved awareness
- create awareness artifacts
- summarize context
- explain evidence
- explain why now
- explain uncertainty
- recommend presentation format

Task Writer may not:

- create facts
- modify graph objects
- modify metrics
- override Continuity Director
- override Awareness QA
- override Trust QA
- create reminders
- perform actions
- decide for the user

## Inputs

Task Writer may consume Continuity Director verdicts:

- PASS
- WATCH
- CONCERN
- HIGH_CONCERN

Task Writer may consume Awareness QA outcomes:

- SURFACE
- PRIORITIZE

Task Writer may consume Trust QA outcomes:

- ALLOW
- ALLOW_WITH_CAUTION

Task Writer may consume GRAPH:

- related entities
- evidence
- context

Task Writer may consume METRICS:

- supporting signals

## Primary Question

How should this be communicated?

If Task Writer cannot explain what, why, and why now, the awareness item must not advance.

## Outputs

Task Writer produces explainable awareness artifacts for future Moon Today.

Outputs are awareness artifacts, not decisions, reminders, notifications, actions, or UI implementations.

## Non-Implementation Boundary

This charter does not define UI, notifications, reminders, actions, storage, ranking, retrieval, or execution behavior.
