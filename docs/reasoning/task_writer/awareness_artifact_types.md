# Awareness Artifact Types v1

## Purpose

This reference defines conceptual awareness artifact types Moon Task Writer may produce.

Examples only. No implementation.

## Awareness Card

Short attention item.

Use when an approved concern can be explained briefly with evidence, why now, confidence, and uncertainty.

## Relationship Watch

Relationship continuity concern.

Use when an approved and permitted relationship concern should be presented carefully, often with visible trust notes.

## Commitment Risk

Commitment continuity concern.

Use when an approved commitment risk affects trust, responsibility, deadline, or follow-through.

## Goal Drift Notice

Goal alignment concern.

Use when an approved goal drift concern explains divergence between intention and effort.

## Project Momentum Alert

Project continuity concern.

Use when an approved project concern indicates possible momentum loss, hidden abandonment, or context decay.

## Learning Continuity Note

Learning concern.

Use when an approved learning concern indicates fragmentation, abandonment, or knowledge decay.

## Weekly Reflection Candidate

Long-form awareness.

Use when an approved concern is better handled as reflective context rather than immediate attention.

## Non-Implementation Boundary

Artifact types are conceptual. They do not define UI layouts, cards, notifications, reminders, ranking, or Today behavior.
