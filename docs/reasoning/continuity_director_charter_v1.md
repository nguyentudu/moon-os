# Continuity Director Charter v1

## Purpose

The Continuity Director is the first reasoning authority in Moon OS.

It exists to answer:

Among everything known about a person's continuity world, what truly deserves attention?

The Continuity Director evaluates continuity significance. It does not build an assistant, agent, memory retrieval system, reminder system, or user interface.

## Core Principle

The Continuity Director does not create memories.

The Continuity Director does not create facts.

The Continuity Director does not make decisions.

The Continuity Director evaluates continuity significance.

## Authority

The Continuity Director may:

- evaluate graph entities
- evaluate metrics
- evaluate continuity signals
- identify continuity concerns
- identify continuity opportunities
- identify continuity risks
- explain reasoning

The Continuity Director may not:

- modify user data
- change graph objects
- change metrics
- override Trust Governance
- override Attention Constitution
- create reminders
- perform actions
- contact users
- make decisions on behalf of users

## Primary Question

For every candidate:

Why does this matter?

If the Director cannot answer, the candidate should not advance.

The Director produces Continuity Verdicts for downstream:

- Awareness QA
- Trust QA
- Task Writer
- Moon Today

Continuity Verdicts are recommendations for review, not decisions.

## Non-Implementation Boundary

This charter does not define agent behavior, memory retrieval, UI, reminders, notifications, storage, graph mutation, metric mutation, or automated action.
