# Continuity Director Review Framework v1

## Purpose

The review framework defines how a candidate continuity signal should be evaluated before it advances beyond the Continuity Director.

This is a review structure only. It is not an agent workflow, UI, retrieval process, or automation.

## Review Questions

### What continuity object is involved?

Identify the graph entity or relationship under review.

Examples:

- Project
- Goal
- Commitment
- Decision
- Memory
- Relationship
- LearningTopic
- Event
- Thread
- Insight

### What evidence exists?

List the source-linked evidence supporting the candidate.

If evidence is missing, the verdict should be UNKNOWN.

### What metrics contributed?

Identify metric indicators that informed the review.

Metrics may contribute but may not decide the verdict.

### What continuity risk exists?

Name the concern type, risk, or opportunity.

If no meaningful continuity risk exists, PASS may be appropriate.

### Why does it matter?

Explain the continuity significance.

If this cannot be answered, the candidate should not advance.

### Why now?

Explain the timing relevance.

If timing is not justified, WATCH, PASS, or UNKNOWN may be appropriate.

### What uncertainty exists?

Expose missing evidence, low confidence, conflicting evidence, trust sensitivity, or attention cost.

### Should this advance?

Choose a Continuity Verdict:

- PASS
- WATCH
- CONCERN
- HIGH_CONCERN
- UNKNOWN

## Review Output

A review should include:

- continuity object
- evidence summary
- metric inputs
- concern classification
- verdict
- explanation
- uncertainty
- trust boundary status
- attention status
- downstream readiness

## Non-Implementation Boundary

This framework does not define storage, retrieval, UI, agent behavior, reminders, notifications, scoring logic, or action execution.
