# Attention Continuity Cases v1

## Scenario A1: Attention Overload

- Scenario Name: Attention Overload
- Context: 15 valid awareness items exist, but only 2 have high awareness value.
- Graph State: Many active concerns; two high-impact commitments; several weak project and learning signals.
- Relevant Metrics: Attention Load high; Signal Density low; Awareness Yield concentrated in two items.
- Expected Continuity Director Verdict: CONCERN
- Expected Awareness QA Outcome: PRIORITIZE top 2, SUPPRESS or DEFER most items
- Expected Trust QA Outcome: ALLOW for low-sensitivity top items
- Expected Task Writer Artifact: Commitment Risk and Project Momentum Alert
- Expected Today Visibility: Only high-value items visible
- Why This Outcome Is Correct: Attention Constitution requires fewer strong signals over many weak signals.

## Scenario A2: Silence Scenario

- Scenario Name: Silence Scenario
- Context: No meaningful continuity concerns today.
- Graph State: Projects stable; goals aligned; commitments fulfilled; no trust-sensitive surfacing.
- Relevant Metrics: Attention Load low; Awareness Yield low; Trust Risk low.
- Expected Continuity Director Verdict: PASS
- Expected Awareness QA Outcome: SUPPRESS
- Expected Trust QA Outcome: ALLOW
- Expected Task Writer Artifact: None
- Expected Today Visibility: No Today items
- Why This Outcome Is Correct: Silence is valid and prevents awareness inflation.
