# Deployment Recommendations v1

## Purpose

Define what should and should not happen after validated intelligence assessment.

## What Should Happen Next

Proceed to Today MVP Proof.

Keep scope constrained to approved awareness artifacts and Today Surface Contract behavior.

Preserve audit traces for any new benchmark execution.

Continue constitutional monitoring.

## What Should Not Happen Next

Do not build ingestion yet.

Do not onboard pilot users yet.

Do not connect external data sources.

Do not add notifications.

Do not add reminders.

Do not add agents.

Do not turn Today into a dashboard or feed.

## What Must Be Monitored

- trust rejection handling
- attention suppression
- evidence requirements
- why-now requirements
- uncertainty visibility
- Task Writer wording
- Today visibility rules

## What Must Be Protected

- consent boundaries
- Fresh Start Mode
- no-old-pull behavior
- silence as valid outcome
- user agency
- no hidden retrieval

## What Evidence Is Still Missing

- live implementation behavior
- adversarial scenario performance
- longitudinal multi-day validation
- human readability review
- internal pilot safety review
- external pilot evidence

## Recommendation

Proceed only to constrained Today MVP Proof.

Do not proceed to data ingestion, internal pilot, or external pilot yet.
