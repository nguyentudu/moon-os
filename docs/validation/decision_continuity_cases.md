# Decision Continuity Cases v1

## Scenario D1: Decision Instability

- Scenario Name: Decision Instability
- Context: Repeated reversals with no preserved rationale.
- Graph State: Decision reversed multiple times; affected Project and Goal; rationale missing.
- Relevant Metrics: Decision Pressure high; Reversal Frequency high; Context Preservation low.
- Expected Continuity Director Verdict: CONCERN
- Expected Awareness QA Outcome: SURFACE
- Expected Trust QA Outcome: ALLOW
- Expected Task Writer Artifact: Awareness Card
- Expected Today Visibility: Visible
- Why This Outcome Is Correct: Repeated reversals without rationale threaten future judgment and project continuity.
