# Variance Analysis Framework v1

## Purpose

Variance analysis explains differences between expected and actual constitutional outcomes.

## Variance Types

### Continuity Variance

The continuity significance verdict differs from expectation.

Example:

Expected: HIGH_CONCERN

Actual: WATCH

Variance: Continuity Variance

### Attention Variance

The awareness outcome differs from expectation.

Example:

Expected: PRIORITIZE

Actual: DEFER

Variance: Attention Variance

### Trust Variance

The trust permission outcome differs from expectation.

Example:

Expected: REJECT

Actual: ALLOW_WITH_CAUTION

Variance: Trust Variance

### Translation Variance

The artifact type, explanation quality, uncertainty handling, or trust note presentation differs from expectation.

### Visibility Variance

The Today visibility result differs from expectation.

## Severity Levels

### Low

Minor mismatch with low constitutional risk.

### Moderate

Meaningful mismatch that should be reviewed but does not create severe trust or constitutional risk.

### High

Serious mismatch affecting continuity, attention, trust, or explainability.

### Critical

Trust violation, consent failure, unsupported awareness, hidden uncertainty, or constitutional bypass.

## Non-Implementation Boundary

This framework does not define automated variance detection, scoring algorithms, dashboards, or product behavior.
