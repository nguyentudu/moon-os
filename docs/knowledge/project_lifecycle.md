# Project Lifecycle v1

## Purpose

Project lifecycle knowledge helps Moon understand how projects begin, move, stall, complete, or fade. It exists so continuity can be preserved even when activity changes over time.

## Core Concepts

- active project: a project with current ownership, meaningful activity, and visible progress.
- dormant project: a project with no recent activity but no clear evidence of abandonment or completion.
- abandoned project: a project where activity, ownership, or intention has effectively stopped without closure.
- completed project: a project whose intended outcome has been reached or explicitly closed.
- ownership: the person or group responsible for keeping the project alive.
- momentum: the degree to which recent activity is moving the project toward its stated purpose.

## Lifecycle

A project may move from proposed to active when ownership and intent become clear. It may remain active while activity and progress continue. It may become dormant when activity pauses without explicit closure. It may become abandoned when dormancy persists and no ownership signal remains. It becomes completed when closure is explicit or the intended outcome is achieved.

Projects may reactivate after dormancy if new activity, renewed ownership, or revised goals appear.

## Signals

- activity: notes, files, meetings, messages, decisions, or artifacts connected to the project.
- progress: visible movement toward project outcomes.
- ownership: named responsibility, repeated initiative, or explicit accountability.
- momentum: cadence, recent changes, and reduction of unresolved work.
- closure language: statements that the project is complete, cancelled, paused, or no longer relevant.

## Risks

- hidden abandonment: the project silently stops while still appearing open.
- false progress: activity happens but does not move the project toward its purpose.
- context loss: rationale, decisions, and next steps become hard to recover.
- stale ownership: responsibility remains assigned after the person has stopped acting on it.

## Continuity Relevance

Projects carry commitments, decisions, relationships, learning, and identity over time. Understanding lifecycle state helps Moon preserve context without treating every old project as equally alive.

## Future Metrics Candidates

- project momentum score
- project dormancy risk
- ownership clarity score
- progress quality score
- context loss risk
- closure confidence
